# Writing PyCS CS Commands

A PyCS command is a class that subclasses `cs.Cmnd` (from `bisos.b import cs`).
Each command declares its parameter shape, argument shape, and a `cmnd()` method
that does the work.

## The class-head dblock is generated — author only the body

The class header — the `class <Name>(cs.Cmnd):` line and the `cmnd()` signature —
is generated by a `####+BEGIN: b:py3:cs:cmnd/classHead ...` dblock. Do NOT
hand-edit the content between `####+BEGIN:` and `####+END:`. Author only the
body *after* the `####+END:` marker.

Example header (generated — do not hand-edit):

```python
####+BEGIN: b:py3:cs:cmnd/classHead :cmndName "myCommand" :extent "verify" :ro "cli" :parsMand "activity" :parsOpt "root templates" :argsMin 0 :argsMax 0 :pyInv ""
""" #+begin_org
*  ... <<myCommand>>  =verify= parsMand=activity parsOpt="root templates" ro=cli
#+end_org """
class myCommand(cs.Cmnd):
    cmndParamsMandatory = [ 'activity', ]
    cmndParamsOptional = [ 'root', 'templates', ]
    cmndArgsLen = {'Min': 0, 'Max': 0,}

    @cs.track(fnLoc=True, fnEntry=True, fnExit=True)
    def cmnd(self,
             rtInv: cs.RtInvoker,
             cmndOutcome: b.op.Outcome,
             activity: typing.Optional[str]=None,
             root: typing.Optional[str]=None,
             templates: typing.Optional[str]=None,
    ) -> b.op.Outcome:

        failed = b_io.eh.badOutcome
        callParamsDict = {'activity': activity, 'root': root, 'templates': templates, }
        if self.invocationValidate(rtInv, cmndOutcome, callParamsDict, None).isProblematic():
            return failed(cmndOutcome)
        activity = csParam.mappedValue('activity', activity)
        root = csParam.mappedValue('root', root)
        templates = csParam.mappedValue('templates', templates)
####+END:
```

The header dblock parameters:

| Dblock parameter | Meaning                                                       |
|------------------|---------------------------------------------------------------|
| `:cmndName`      | Class name and CLI verb                                       |
| `:extent`        | Usually `"verify"` — enables invocation-validation prologue   |
| `:ro`            | Remote-ops mode: `"cli"`, `"noCli"`, or `"both"`              |
| `:parsMand`      | Space-separated mandatory parameter names                     |
| `:parsOpt`       | Space-separated optional parameter names                      |
| `:argsMin` / `:argsMax` | Positional argument count bounds                       |
| `:pyInv`         | Python-invocation signature (usually empty)                   |

Update the dblock header when you change parameters/args; Blee/Emacs
regenerates the body when the file is opened. Outside Blee, use
`py-dblock.cs` (from `bisos.pyDblock`) to expand dblocks.

## The `cmnd()` body

Everything *after* `####+END:` is the command implementation. Standard shape:

```python
        self.cmndDocStr(f""" #+begin_org
** [[elisp:(org-cycle)][| *CmndDesc:* | ]]  Short description of what this command does.
Optionally multi-line, in org-mode.
        #+end_org """)

        # -- do the work --
        # Read parameters via the local variables (activity, root, templates)
        # Report progress via b_io.ann.note(...)
        # Report user-error via b_io.eh.problem_usageError(...) + return failed(cmndOutcome)

        return cmndOutcome.set(
            opError=b.op.OpError.Success,
            opResults=f"Completed for activity={activity}",
        )
```

Key pieces of the body:

- **`self.cmndDocStr(...)`** — the org-mode description. First thing in the
  body. This is what `-i <cmndName>` prints as documentation.
- **`self.captureRunStr(...)`** — *optional*. Captures expected example output
  inline for that command. This is BISOS's inline unit test — see the
  `dblock-authoring` skill for the mechanics.
- **Return value** — always `cmndOutcome.set(...)` on success, or
  `failed(cmndOutcome)` on error. Never `return None` and never raise.

## Parameters vs. arguments

- **Parameters** (`parsMand` / `parsOpt`) — named, passed on the CLI as
  `--parName=value`. Declared in the enclosing csxu's `commonParamsSpecify()`
  (or via `parPermanence="userConfig"` from `bisos.b.userConfig_csu`).
  Read them as local variables in the `cmnd()` signature.

- **Arguments** (`argsMin` / `argsMax`) — positional, passed after the verb.
  When the command takes args, also declare a `cmndArgsSpec()` method (its
  own `b:py3:cs:method/args` dblock) and read args in `cmnd()` via
  `self.cmndArgsGet(...)`.

Example command taking positional args:

```python
    cmndArgsLen = {'Min': 1, 'Max': 9999,}

    # ... cmnd() body ...
    def cmndArgsSpec(self):
        cmndArgsSpecDict = cs.CmndArgsSpecDict()
        cmndArgsSpecDict.argsDictAdd(
            argPosition="0&-1",
            argName="filePaths",
            argChoices=[],
            argDescription="One or more file paths",
        )
        return cmndArgsSpecDict
```

## Register the command in the `examples` menu

Every command should be visible in the csxu's `examples` output — this is
how users discover it. In the csxu's `examples` command body:

```python
cs.examples.menuChapter('=myCommand= -- one-line description')
cmnd('myCommand',
     pars=od([('activity', 'bisos-pip')]),
     comment="# Install bisos-pip templates in current directory")
```

Group related commands under a `menuChapter`. For CSU files (`_csu.py`),
register via the CSU's own `examples_csu()` function; the parent csxu will
call it via `pyCmnd()`.

## Common patterns

- **Read a persistent user-config param**: use
  `userConfig_csu.parGet('paramName', localOverride)` from
  `bisos.b.userConfig_csu`. Local CLI override wins if provided.

- **Emit user-visible progress**: `b_io.ann.note(f"Message")`. These are
  informational messages, distinct from `opResults`.

- **Report user error**: `b_io.eh.problem_usageError("Message")` then
  `return failed(cmndOutcome)`. Do *not* raise Python exceptions for user
  errors.

- **Determine target directory** (repo vs. cwd): follow the existing
  pattern in `initiate` — subprocess to `git rev-parse --show-toplevel` for
  `root=repo`, else `pathlib.Path.cwd()`.

## What NOT to do

- Do NOT hand-edit content between `####+BEGIN:` and `####+END:` — regenerate
  the dblock instead.
- Do NOT `return None`, `return 0`, or `raise` in `cmnd()` — always return a
  set `cmndOutcome`.
- Do NOT swallow errors silently — use `b_io.eh.problem_usageError` and
  return `failed(cmndOutcome)`.
- Do NOT add a new command without registering it in the csxu's `examples`.
- Do NOT define parameters inline in the command class — declare them in
  the csxu's `commonParamsSpecify()` so the framework wires them into
  argparse. The command class only *lists* which params it accepts.
