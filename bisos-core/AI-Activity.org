* General: Commands

All commands assume you are in =py3/=. For build, install, and publish, see
*bisos-pip Build System* above.

- *Lint / type-check* (paths from the in-file "Workbench" headers): the venv
  interpreter is =/bisos/venv/py3/bisos3/bin/python=; linters live in
  =/bisos/pipx/bin/= (=pyflakes=, =flake8=, =pylint=, =pycodestyle=). =mypy= is
  used (a =.mypy_cache/= for py3.11 is present).
- *Smoke test* (no test suite exists): run a seed's example menu, e.g.
  =bin/addCmnds-helloWorld.pcs= (prints the example menu) or
  =bin/addCmnds-helloWorld.pcs -i helloWorld= (runs the =helloWorld= command).

There is no unit-test framework in this repo. Verification is done by running
the =.cs=/=.pcs= executables and inspecting their output --- the expected output
is captured inline in each command via =self.captureRunStr(...)=.

Some execution examples are at:
@/bisos/git/bxRepos/bisos-pip/gitist/py3/panels/bisos.gitist/_nodeBase_/fullUsagePanel-en.org

* General: Writing CS commands

Commands are classes subclassing =cs.Cmnd= (from =bisos.b import cs=). Each
declares =cmndParamsMandatory=, =cmndParamsOptional=, =cmndArgsLen=, and a
=cmnd()= method that does the work. The gitist commands live in
=py3/bisos/gitist/gitist_csu.py= --- read =whichReactFramework= and
=placeHolder= there as templates.

The class header (the =class <Name>(cs.Cmnd):= line and the =cmnd()= signature)
is generated from a =####+BEGIN: b:py3:cs:cmnd/classHead ...= dblock --- do not
hand-edit between the =BEGIN=/=END= markers. Author only the body *after* the
=####+END:= marker:

- =cmndParamsMandatory= / =cmndParamsOptional= :: lists of parameter names the
  command accepts (declared on the =classHead= dblock as =:parsMand= / =:parsOpt=).
- =cmndArgsLen= :: =({'Min': N, 'Max': M})= positional-argument bounds. When the
  command takes args, also define a =cmndArgsSpec()= method (its own
  =b:py3:cs:method/args= dblock) using =cs.CmndArgsSpecDict().argsDictAdd(...)=,
  and read them in =cmnd()= with =self.cmndArgsGet(...)=.
- =cmnd(self, rtInv, cmndOutcome, ...)= :: the implementation. Start by calling
  =self.cmndDocStr(...)= for the org description, optionally
  =self.captureRunStr(...)= to capture expected example output (this *is* the
  test --- see *Commands* above), do the work, and return
  =cmndOutcome.set(opError=b.OpError.Success, opResults=...)=.

Register the command in the seed's example menu (=examples_csu= /
=examples_seed= in the same file) so it shows up in the plant's example output.
